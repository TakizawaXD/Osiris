# Redactor de Contenido y Copywriter

Esta habilidad capacita al agente para redactar con excelente ortografía, tono adecuado y claridad cualquier tipo de texto en español dirigido a usuarios finales o desarrolladores.

## Cuándo usar esta habilidad

Activa esta habilidad cuando debas:
- Redactar microcopys de interfaz (textos de botones, placeholders, mensajes de éxito o error).
- Escribir la documentación técnica del proyecto (README.md, guías de instalación, documentación de APIs).
- Crear flujos de onboarding, correos electrónicos transaccionales o artículos de blog.

## Cómo usarla: Reglas de Escritura y Tono

### 1. Tono y Voz de la Marca
Adapta el estilo de comunicación según el tipo de aplicación:
- **Profesional / Corporativo (Fintech, Legal, Enterprise)**: Tono formal pero directo. Uso de la tercera persona u "nosotros" institucional. Evitar tecnicismos excesivos sin explicación.
- **Cercano / Juguetón (SaaS creativos, Apps de Productividad, Educación)**: Uso del tuteo ("tú" o "vos"). Tono inspirador, amigable y motivador.
- **Técnico (APIs, Herramientas de Desarrollo)**: Tono pragmático, preciso y directo. Enfoque en ejemplos de código claros y explicaciones concisas.

### 2. Microcopy de Interfaz (UI Copy)
- **Mensajes de Error Empáticos y Resolutivos**: Nunca digas simplemente "Ha ocurrido un error". Un buen mensaje de error debe constar de:
    - Qué pasó (en lenguaje sencillo, no códigos técnicos).
    - Por qué pasó (si es posible determinarlo).
    - Cómo solucionarlo (un botón de reintento, enlace a soporte o sugerencia clara).
- **Llamadas a la Acción Claras (CTA)**: Los botones deben describir la acción que realiza el usuario (ej. "Crear cuenta gratuita" en lugar de "Enviar").

### 3. Documentación Técnica Profesional
Asegura que toda documentación siga esta estructura básica:
1. **Introducción**: Qué hace la herramienta y qué problema resuelve.
2. **Requisitos Previos**: Software o credenciales necesarias.
3. **Instalación rápida**: Comandos de consola limpios y listos para copiar.
4. **Ejemplos de Uso**: Fragmentos de código mínimos y funcionales.
5. **Referencia de API**: Tabla con parámetros, tipos de datos y respuestas.
